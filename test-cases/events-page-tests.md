# Events Page Test Cases

## TC-01. Перевірка відображення списку подій і відкриття картки

**Title:** Перевірка відображення списку подій і відкриття картки події

**Preconditions:**
- Користувач відкрив сторінку https://www.greencity.cx.ua/#/greenCity/events
- Сторінка завантажена повністю

**Test Steps:**

| Step | Action | Data | Expected Result |
|---|---|---|---|
| 1 | Перевірити, що сторінка Events відкрилася | - | Відображається заголовок сторінки, фільтри та список подій |
| 2 | Обрати будь-яку подію зі списку | Натиснути на картку або кнопку More | Відкривається детальна інформація про подію |
| 3 | Перевірити основні дані в картці | Назва, дата, час, локація, статус | Дані відображаються коректно та відповідають обраній події |

## TC-02. Фільтрація подій за параметрами

**Title:** Перевірка роботи фільтрів подій

**Preconditions:**
- Користувач відкрив сторінку https://www.greencity.cx.ua/#/greenCity/events
- На сторінці доступний список подій

**Test Steps:**

| Step | Action | Data | Expected Result |
|---|---|---|---|
| 1 | Відкрити фільтр типу події | Тип події | Відображається список доступних типів |
| 2 | Обрати один тип події | Наприклад, Economic | Список подій оновлюється та показує лише події вибраного типу |
| 3 | Додати ще один фільтр | Наприклад, дата або статус | Результати звужуються відповідно до обраних критеріїв |
| 4 | Очистити фільтри | Reset all | Відображається повний список подій без активних фільтрів |

## TC-03. Створення події з обов'язковими полями

**Title:** Перевірка форми створення події та валідації обов'язкових полів

**Preconditions:**
- Користувач має доступ до функції створення події
- Відкрита сторінка Events

**Test Steps:**

| Step | Action | Data | Expected Result |
|---|---|---|---|
| 1 | Натиснути кнопку Create event | - | Відкривається форма створення події |
| 2 | Спробувати зберегти форму без обов'язкових даних | Назва, дата, час, опис | Система показує повідомлення про помилки валідації |
| 3 | Заповнити обов'язкові поля коректними даними | Назва, дата, час, локація, тип події | Форма приймає дані без помилок |
| 4 | Підтвердити створення | Publish | Подія успішно створюється та з'являється у списку |

## TC-05. Негативна перевірка створення події без обов'язкових полів

**Title:** Перевірка помилок валідації при створенні події без обов'язкових даних

**Preconditions:**
- Користувач відкрив сторінку https://www.greencity.cx.ua/#/greenCity/events
- Користувач має доступ до форми створення події

**Test Steps:**

| Step | Action | Data | Expected Result |
|---|---|---|---|
| 1 | Натиснути кнопку Create event | - | Відкривається форма створення події |
| 2 | Не заповнювати обов'язкові поля | Назва, дата, час | Система не дозволяє зберегти форму |
| 3 | Спробувати підтвердити створення | Publish | Відображаються повідомлення про помилки валідації |
| 4 | Перевірити результат | - | Подія не створюється і не з'являється у списку |
